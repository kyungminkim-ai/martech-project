# 앱푸시 메시지 작성 정책 (Writing Policy)

> 이 문서는 앱푸시 메시지의 **구체적인 작성 규칙**을 정의합니다.
> 브랜드 아이덴티티(Voice & Tone, 금칙어)는 `brand_guidelines.md` 참조.
> 파이프라인 절차(Step별 처리)는 `message_policy.md` 참조.

---

## 1. 공통 포맷 규칙

### 광고 표기 (법적 의무)
- 모든 앱푸시 본문(contents)은 반드시 `(광고) `로 시작
- 제목(title)에는 광고 표기 불필요

### 수신거부 문구 (고정)
```
수신거부 : 메인 상단 알림 > 설정 > 알림 OFF
```
- 모든 본문 마지막에 `\n`으로 구분하여 삽입
- **LLM이 생성하지 않음** — Python 후처리(`append_unsubscribe`)로 자동 추가

### 브랜드명 표기
- 반드시 `brand_list.csv`의 `brand_nm` 사용
- 영문 브랜드도 `brand_nm`이 있으면 한국어 표기 우선
- 브랜드명 임의 번역 금지

## 2. 종결 어미 규칙 (전체 공통)

### ✅ 허용 — 명사형 종결
| 유형 | 예시 |
|------|------|
| 혜택 | `...혜택`, `...할인`, `...증정`, `...쿠폰`, `...적립금` |
| 발매/출시 | `...발매`, `...드롭`, `...출시`, `...에디션`, `...컬렉션` |
| 상태 | `...진행`, `...마감`, `...수량`, `...존재감` |

### ❌ 금지 — 동사형·형용사형 종결
| 유형 | 예시 |
|------|------|
| 동사형 명령 | `서둘러요`, `해보세요`, `받아보세요`, `만나보세요`, `느껴보세요` |
| 동사형 명령 | `확인하세요`, `경험해보세요`, `가세요`, `오세요` |
| 명령조 | `사세요!`, `하세요!`, `마세요!`, `놓치지 마세요!` |

## 3. 제목 (title)

| 항목 | 기준 |
|------|------|
| 길이 | 15~40자 (공백 포함) |
| 구조 | 브랜드명/상품명을 앞부분에 배치 |
| 종결 | 명사형 (`발매`, `드롭`, `출시`, `할인`, `에디션`) |
| 금지 | 감탄사, 명령조, 의미 없는 코드, `(광고)` 표기 |

### 소재 유형별 패턴
| 유형 | 패턴 | 예시 |
|------|------|------|
| 콜라보/협업 | `브랜드A × 브랜드B [설명]` | 컨버스 × 매드해피 척 70 |
| 콜라보 무신사 에디션 | `브랜드 × 브랜드, 무신사 에디션` | 레이브 × 헬로키티, 무신사 에디션 |
| 아티스트/IP 드롭 | `브랜드명 무신사 드롭` | 킥플립 무신사 드롭 |
| 단독 선발매 | `브랜드 무신사 단독 선발매` | 시에라디자인 무신사 단독 선발매 |
| 시즌 신상 | `브랜드 26SS/25FW 컬렉션명` | 겐조 26SS 카툰 컬렉션 |
| 단순 할인 | `브랜드 최대 N% 할인` | 리드볼트 최대 20% 할인 |
| 대형 할인 캠페인 | `[최대 N%] 이벤트명` | [최대 80%] 티셔츠 페스티벌 오픈 |
| 사은품/기획세트 | `브랜드 구매 시 [사은품] 증정` | 일리윤 기획세트 구매 시 몬치치 증정 |

## 4. V1 BENEFIT — 혜택 강조형

**목적:** 할인·쿠폰·적립금 등 수치적 혜택을 명확히 전달

| 항목 | 기준 |
|------|------|
| 시작 | 반드시 `(광고) ` |
| 길이 | 40~60자 (수신거부 제외) |
| 핵심 | 혜택 수치 강조 (%, 원 단위 명확히) |
| 연결 | 복수 혜택 `+`로 연결 (최대 3개) |
| 종결 | 명사형 필수 |

**좋은 예시:**
- `(광고) 최대 30% 쿠폰 + 최대 5만원 적립금 + 최대 1만원 결제혜택`
- `(광고) 최대 80% 할인 + 단 하루 25% 쿠폰`
- `(광고) 최대 31% 할인 + 쿠폰 + 사은품 혜택`

**나쁜 예시:**
- `(광고) 한정판 품절 전 서둘러요` ← 동사형 종결
- `(광고) 지금 바로 혜택 받아보세요` ← 동사형 종결

## 5. V2 BRAND — 브랜드 감성형

**목적:** 브랜드의 이야기·차별성을 감성적으로 전달

| 항목 | 기준 |
|------|------|
| 시작 | 반드시 `(광고) ` |
| 길이 | 25~45자 (수신거부 제외) |
| 핵심 | 브랜드/제품 고유성·차별성 감성 표현 |
| 수치 | 최소화 (혜택 나열 금지 — V1과 중복 방지) |
| 종결 | 명사형 필수 |

**좋은 예시:**
- `(광고) 올 블랙보다 시크한 그레이 스티치의 존재감`
- `(광고) 다시 시작되는 그들의 이야기를 담은 굿즈, 무신사 드롭 발매`

**나쁜 예시:**
- `(광고) 현진과 에스에스알엘의 협업, 지금 만나보세요` ← 동사형 종결
- `(광고) 스타일을 느껴보세요` ← 동사형 종결

## 6. V3 BEST — V1+V2 합성 최선책

**목적:** V1과 V2를 비교·분석하여 각 장점을 결합한 최선의 메시지 생성

| 항목 | 기준 |
|------|------|
| 시작 | 반드시 `(광고) ` |
| 길이 | 25~45자 (수신거부 제외) |
| 핵심 | V1의 혜택 정보 + V2의 감성 표현을 균형 있게 결합 |
| 중복 금지 | **제목(title)에 쓰인 내용을 본문에서 그대로 반복 금지** |
| 종결 | 명사형 필수 |

**생성 로직:**
1. V1 분석 — 제목과 본문이 중복되는가? 혜택 수치가 명확한가?
2. V2 분석 — 감성 표현이 적절한가? 클릭 동기가 충분한가?
3. 각각의 약점을 보완하여 가이드라인을 모두 충족하는 메시지 생성

**예시 (합성 프로세스):**
```
제목:  에스에스알엘 X 현진 무신사 에디션
V1:    (광고) 에스에스알엘 × 현진 무신사 에디션, 최대 31% 할인 + 쿠폰 + 사은품 혜택
       → 약점: 제목 내용과 본문 중복
V2:    (광고) 에스에스알엘과 현진이 함께 완성한 무신사 에디션
       → 약점: 혜택 정보 전무
V3:    (광고) 현진 × 에스에스알엘, 최대 31% 할인 + 사은품 혜택
       → 제목 중복 제거 + 핵심 혜택 유지
```

**발송 우선순위:** V3 > V1 > V2 (V3가 항상 `contents`로 사용됨)
