# 앱푸시 메시지 작성 정책 (Writing Policy)

> 이 문서는 앱푸시 메시지의 **구체적인 작성 규칙**을 정의합니다.
> 브랜드 아이덴티티(Voice & Tone, 금칙어)는 `brand_guidelines.md` 참조.
> 파이프라인 절차(Step별 처리)는 `message_policy.md` 참조.

---

## 1. 공통 포맷 규칙

### 광고 표기 (법적 의무)
- 모든 앱푸시 본문(contents)은 반드시 `(광고) `로 시작
- 제목(title)에는 광고 표기 불필요

### 수신거부 문구 (고정)
```
수신거부 : 메인 상단 알림 > 설정 > 알림 OFF
```
- 모든 본문 마지막에 `\n`으로 구분하여 삽입
- **LLM이 생성하지 않음** — Python 후처리(`append_unsubscribe`)로 자동 추가

### 브랜드명 표기
- 반드시 `brand_list.csv`의 `brand_nm` 사용
- 영문 브랜드도 `brand_nm`이 있으면 한국어 표기 우선
- 브랜드명 임의 번역 금지

## 2. 종결 어미 규칙 (전체 공통)

### ✅ 허용 — 명사형 종결
| 유형 | 예시 |
|------|------|
| 혜택 | `...혜택`, `...할인`, `...증정`, `...쿠폰`, `...적립금` |
| 발매/출시 | `...발매`, `...드롭`, `...출시`, `...에디션`, `...컬렉션` |
| 상태 | `...진행`, `...마감`, `...수량`, `...존재감` |

### ❌ 금지 — 동사형·형용사형 종결
| 유형 | 예시 |
|------|------|
| 동사형 명령 | `서둘러요`, `해보세요`, `받아보세요`, `만나보세요`, `느껴보세요` |
| 동사형 명령 | `확인하세요`, `경험해보세요`, `가세요`, `오세요` |
| 명령조 | `사세요!`, `하세요!`, `마세요!`, `놓치지 마세요!` |

> **자동 감지·수정**: Pipeline 3이 동사형 종결을 감지하면 LLM 재호출로 자동 수정한다.

## 3. 제목 (title)

| 항목 | 기준 |
|------|------|
| 길이 | 5~40자 (공백 포함) |
| 행동어 | 발매·출시·드롭·단독·론칭은 원칙적으로 **제목에 쓰지 않음** — 단, 드롭·발매 자체가 소재의 핵심 정체성인 경우(신규 발매, 단독 드롭, 한정 론칭 등)는 허용 |
| 이모지 | 금지 |
| 금지 | 감탄사, 명령조, 의미 없는 코드, `(광고)` 표기 |

### 콜라보/협업 소재 — 강제 규칙

이벤트명 또는 원본 타이틀에 "X / x / ×" 패턴이나 "with" 패턴이 있으면 콜라보 소재로 판단하며,
제목에 반드시 두 브랜드/아티스트 이름을 모두 포함한다.

**콜라보 제목 허용 형식은 반드시 아래 2가지 중 하나:**
- `Brand A x Brand B` — x, ×, X 패턴이 원본에 있을 때 (x는 소문자)
- `Brand A with Brand B` — with 패턴이 원본에 있을 때 (with는 소문자)

| 올바른 제목 | 잘못된 제목 | 문제 |
|------------|------------|------|
| `알리스 x 오정규` | `오정규 협업 발매` | 알리스 누락 + 행동어 |
| `수아레 x 너드킹` | `수아레 x 너드킹 발매` | 행동어 "발매" 포함 |
| `서브웨이 with DAY6` | `서브웨이 X DAY6` | 원본이 with인데 x로 변경 |
| `마크모크 x 태리태리 써머 슈즈` | `무신사 단독 발매` | 콜라보 쌍 완전 누락 |
| `여밈 x 미세키서울 무신사 에디션` | `여밈 x 미세키서울, 무신사 단독 콜라보 발매` | 행동어 + 과잉 |

콜라보 쌍 뒤에 상품명·에디션명을 붙이는 것은 허용:
- `더마토리 x 톡신 공동개발 블랙세럼` ← OK
- `에스에스알엘 x 현진 무신사 에디션` ← OK

### `with` 패턴 콜라보 — 처리 규칙

`Brand A with Brand B` 형식은 원본(bizest_raw `main_title` 또는 `event_name`)에서 "with"가 사용된 경우에만 그대로 보존한다.
구분자 형식은 원본을 따르며, 대상 유형(브랜드/아티스트 구분)에 따른 별도 전환 없이 통일 처리한다.

| 예시 | 처리 |
|------|------|
| `서브웨이 with DAY6` | 원본이 with → 제목: `서브웨이 with DAY6` ✅ |
| `나이키 X 스톤아일랜드` | 원본이 X → 제목: `나이키 x 스톤아일랜드` ✅ |
| `글랙 × 지꾸` | 원본이 × → 제목: `글랙 x 지꾸` ✅ |

### 비콜라보 소재

브랜드명·상품명 중심 **또는** 호기심 유발 훅 문구. 두 가지를 섞으면 본문과 중복된다.

| 유형 | 패턴 | 예시 |
|------|------|------|
| 단독/시즌 신상 | `브랜드 [상품/시즌명]` | 시에라디자인 25FW 컬렉션 |
| 단순 할인 | `브랜드 최대 N% 할인` | 리드볼트 최대 20% 할인 |
| 대형 할인 캠페인 | `[최대 N%] 이벤트명` | [최대 80%] 티셔츠 페스티벌 오픈 |
| 훅 필요 시 | 감성·호기심 유발 문구 | 빵처럼 맛있는 쉐이크 발견 |

### main_title 40자 초과 처리
bizest_raw의 `main_title`이 40자를 초과하고 쉼표로 구분된 경우:
- 쉼표 앞 부분(훅)을 제목으로 사용
- 예) `"빵처럼 맛있는 쉐이크 발견, 테이크핏 브레드밀 단독 출시"` → 제목: `"빵처럼 맛있는 쉐이크 발견"`
- 분리 불가능하거나 앞 부분이 5자 미만이면 LLM이 재생성

## 4. 본문 (contents)

**목적:** 제목의 서술어. 제목과 이어 읽었을 때 하나의 완성된 문장이 된다.

| 항목 | 기준 |
|------|------|
| 시작 | 반드시 `(광고) ` |
| 길이 | 25~60자 (수신거부 제외) |
| 혜택 | 할인율·쿠폰·적립금 등 수치가 있으면 반드시 포함 |
| 감성 | 수치 혜택이 없으면 브랜드·제품 차별성 감성 표현 |
| 종결 | 명사형 필수 |

### 제목-본문 중복 금지 (핵심)

제목이 전달한 단어는 본문에서 반복하지 않는다.

**콜라보 소재의 경우:**
- 제목의 "Brand X Brand" 쌍은 본문에서 절대 반복하지 않는다.
- 각 브랜드/아티스트 이름도 금지.
- 본문은 발매·혜택·기간·긴급성만 서술한다.

| 제목 | 올바른 본문 | 잘못된 본문 |
|------|-----------|-----------|
| `알리스 x 오정규` | `(광고) 무신사 단독 한정 발매` | `(광고) 알리스 x 오정규 콜라보, 무신사 단독 발매` |
| `수아레 x 너드킹` | `(광고) 무신사 단독 발매 스트리트 컬렉션` | `(광고) 수아레 x 너드킹 콜라보, 무신사 단독 발매` |
| `더마토리 x 톡신 공동개발 블랙세럼` | `(광고) 4/27~29 단 3일 선론칭` | `(광고) 더마토리 x 톡신 블랙세럼 4/27 선론칭` |

**비콜라보 소재의 경우:**
- 제목의 3자 이상 토큰은 본문 작성 시 금지 단어로 주입됨.

> **자동 감지·수정**: Pipeline 3이 제목-본문 중복을 감지하면 LLM 재호출로 자동 수정한다.

### 복수 혜택 나열

혜택이 2개 이상인 경우 본문에 나열할 수 있다.

| 항목 | 기준 |
|------|------|
| 최대 개수 | 3개 이내 |
| 우선순위 | 할인율 > 쿠폰 > 증정 > 기타 |
| 구분자 | `+` (중복 적용 혜택) 또는 `,` (병렬 나열) |

예) `(광고) 최대 80% 할인 + 최대 30% 쿠폰 + 에디션 1등 상품 특가`

### 소재 원문 컨텍스트 활용

본문 생성 시 아래 소재 원문 필드를 모두 참조한다.

| 소재 필드 | 활용 방식 |
|---------|---------|
| `promotion_content` | 혜택·조건 정보 — 본문 생성의 1차 소스 |
| `remarks` (비고) | 마케터 메모 — 특별 강조 사항이 있으면 본문에 반영 |

단, `remarks`에 `취소`, `CANCEL` 등 필터 키워드만 있는 경우 무시.

## 5. 소재 유형별 본문 규칙

### 콘텐츠형 소재 (랭킹·리포트)

`content_type = "콘텐츠"` 또는 `main_title`에 `랭킹`, `리포트`, `트렌드` 포함 시.

할인·쿠폰 없이 콘텐츠 가치를 전달하는 소재. 감성 표현 대신 정보 요약 중심으로 작성한다.

| 항목 | 기준 |
|------|------|
| 방향 | 감성 표현 ❌ — 콘텐츠 정보 요약 ✅ |
| 패턴 | `[카테고리/분야] [핵심 정보]` |

예)
- 제목: `랭킹 리포트로 보는 12월 트렌드`
- 본문: `(광고) 패션/아웃도어/뷰티 가장 사랑받은 브랜드 TOP 10`

### 이벤트 참여형 소재 (무료참여·퀴즈·경품)

`main_title` 또는 `promotion_content`에 `무퀴즈`, `퀴즈`, `0원`, `무료`, `경품` 포함 시.

할인율 없이 참여 자체가 혜택인 소재. 참여 조건과 받을 수 있는 혜택을 함께 전달한다.

| 항목 | 기준 |
|------|------|
| 방향 | 참여 행동 + 결과 혜택 명시 |
| 패턴 | `[참여 방식] [혜택]` |

예)
- 제목: `겨울에 스키에 0원 찬스`
- 본문: `(광고) 무료 참여만 해도 아메리카노·스키에 교환권 0원`
