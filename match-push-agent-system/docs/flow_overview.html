1. 시스템 목적

[앱푸시 발송 운영] 시트(비제스트 RAW)에 등록된 소재 요청을 받아, 담당자가 Braze에 직접 등록할 수 있는 캠페인메타엔진 운영 시트 형식 CSV를 자동 생성합니다.

입력
[앱푸시 발송 운영] 시트
bizest_raw.csv
출력
캠페인메타엔진 운영 시트 형식
campaign_meta_YYYYMMDD_*.csv
역할 범위
소재 선별 → 메시지 생성 → 검수 플래그
역할 외
Braze 발송
담당자가 직접 시트 확인 후 등록

4. 파이프라인 상세

Pipeline 1 — 소재 선별

비제스트 RAW에서 발송 가능한 소재만 추출합니다.

단계조건처리
① 취소 필터비고란(remarks)에 취소, CANCEL, cancel 포함제외
② 중복 제거landing_url + sourceBrandId + send_dt 조합 중복ID 오름차순 첫 번째만 유지
③ 랜딩 오픈 검증release_start_date_time < send_dt 11:00 미충족제외
④ 마케팅팀 예외register_team_name에 전사마케팅 또는 카테고리마케팅 포함③ 조건 무관하게 선별

중간 결과가 data/pipeline1_output_YYYYMMDD.csv에 저장되므로, Pipeline 2만 재실행할 때 활용할 수 있습니다.

Pipeline 2 — 메타데이터 & 메시지 생성

선별된 소재에 대해 캠페인메타엔진 운영 시트의 컬럼을 채웁니다.

컬럼생성 방식
send_time고정 11:00
target팀명 기반: 여성팀→여성, 남성팀→남성, 그 외→전체
priority전사마케팅→1, 카테고리마케팅→2, 그 외→3
ad_codeAPSCMCD + BASE36 순번 (이전 코드에서 +1 채번)
content_typeURL 패턴: /campaign/→캠페인, /content/→콘텐츠
push_urllanding_url?utm_source=app_push&utm_medium=cr&utm_campaign={ad_code}
title원본 15~40자 + 키워드 충족 시 원본 사용, 그 외 LLM 재생성
contents (V1)LLM — (광고) 시작, 40~60자, 혜택 수치 강조
[검수용] contents_v2 (V2)LLM — (광고) 시작, 25~45자, 브랜드/감성

Pipeline 3 — 검수 검증 (Validation QA)

발송 전 문제가 될 수 있는 항목을 객관적으로 검증합니다. 행을 제거하지 않고 [검수용] 컬럼에 이슈를 기록합니다.

#검증 항목이슈 코드수준
1필수 필드 누락 (title, contents, landing_url, ad_code)*_missing⛔ 오류
2title 길이 범위 (15~40자)title_length_N chars⚠️ 검수
3(광고) 접두어 누락missing_(광고)_prefix⚠️ 검수
4수신거부 문구 누락missing_unsubscribe_text⚠️ 검수
5push_url UTM 파라미터 누락push_url_missing_utm⚠️ 검수
6push_url UTM campaign ≠ ad_codepush_url_campaign_mismatch⚠️ 검수
7contents에 0% 표기 (최소 5% 이상 규칙)zero_percent_in_contents⚠️ 검수
8landing_url https 미적용landing_url_not_https⚠️ 검수
9ad_code 중복 (같은 실행 내)ad_code_duplicate⚠️ 검수
10brand_id 누락brand_id_missing⛔ 오류
11LLM confidence 임계값 미달 (기준: 3.0)low_confidence_v1/v2(N)⚠️ 검수
12title LLM 생성 실패 (fallback)title_source_fallback⚠️ 검수

5. 출력 파일 구조

산출물 경로: output/campaign_meta_YYYYMMDD_HHmmss.csv

컬럼내용자동 생성

contents푸시 본문 V1 혜택강조✅ LLM

6. 실행 방법

사전 준비 (최초 1회)

# 1. 디렉터리 이동
cd match-push-agent-system

# 2. .env 파일 생성 후 API 키 입력
cp .env.example .env
# → .env 파일을 열어 ANTHROPIC_API_KEY=sk-ant-... 입력

# 3. 패키지 설치
pip3 install -r requirements.txt

매일 실행

# 내일 날짜 자동 사용 (기본값)
python3 scripts/run.py

# 발송일 지정
python3 scripts/run.py --date 2026-05-01

# 입력 파일 직접 지정
python3 scripts/run.py --date 2026-05-01 --input input/my_raw.csv

입력 파일 업데이트

매 실행 전 최신 파일로 교체하세요:

파일경로업데이트 주기

브랜드 목록input/brand_list.csv변경 시만

7. 검수 가이드

실행 결과 리포트 예시

============================================================
[push-campaign 완료] send_dt=2026-05-01
============================================================

📊 처리 결과:
선별 소재: 29건
LLM 생성 성공: 29건
오류: 0건
검수 필요: 3건

📁 산출물:
output/campaign_meta_20260501_143022.csv

⚠️ 검수 필요 항목 (id): [1023, 1045, 1089]
============================================================

확인 순서

순서확인 컬럼조치
① 오류 행 우선[검수용] error_flag = True내용 누락 → 수동 작성 필요
② 검수 필요 행[검수용] needs_review = True[검수용] validation_notes 내용 검토
③ V2 문구 비교[검수용] contents_v2V1과 비교 후 더 나은 문구를 contents에 수동 복사

자주 나오는 이슈 코드

이슈 코드원인조치
title_missingLLM API 오류로 제목 생성 실패수동 제목 작성
title_length_N charsLLM이 범위 밖 제목 생성직접 수정 또는 삭제
zero_percent_in_contents할인율 0% 표기 (정책 위반)내용 수정 또는 행 제외
low_confidence_v1(N)LLM 신뢰도 낮음 (기준 3.0 미달)문구 직접 검토 및 수정
title_source_fallback원본과 LLM 제목 모두 부적합제목 수동 작성
